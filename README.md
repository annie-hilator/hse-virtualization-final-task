# Итоговое задание по дисциплине «Основы виртуализации и контейнеризации»
## Описание

В рамках работы была использована виртуальная машина с Ubuntu 22.04.5 LTS, внутри которой выполнена настройка базовых средств безопасности, установлен Docker и развернут набор сервисов с помощью Docker Compose.

Используются три контейнера:
-   nginx (reverse proxy);
-   веб-приложение whoami;
-   PostgreSQL в качестве системы управления базой данных.

Доступ к приложению осуществляется через nginx по адресу:

```
http://192.168.0.97
```

## Виртуальная машина

Виртуальная машина была создана в Oracle VirtualBox.

Параметры виртуальной машины:

-   Операционная система: Ubuntu 22.04.5 LTS
-   Виртуализация: Oracle VirtualBox
-   Архитектура: x86-64
-   Процессор: 2 vCPU
-   Оперативная память: 4 ГБ
-   Размер диска: 20 ГБ
-   Пользователь: user
-   IP-адрес: 192.168.0.97

Пользователь `user` не является пользователем root и имеет права sudo.

### Настройка межсетевого экрана

Для настройки firewall использовался UFW.

Были разрешены только необходимые порты:

```
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw enable
```

Проверка:

```
sudo ufw status
```

Результат:

```
22/tcp ALLOW
80/tcp ALLOW
```

## Состав сервисов

В compose-проекте используются следующие сервисы:

-   `proxy` - контейнер с nginx;
-   `app` - контейнер с приложением whoami;
-   `db` - контейнер с PostgreSQL.

Также дополнительно настроены:

-   отдельная сеть Docker для взаимодействия контейнеров;
-   том Docker для хранения данных PostgreSQL;
-   автоматический перезапуск контейнеров nginx и PostgreSQL;
-   проверка работоспособности PostgreSQL (healthcheck).

## Проверка сохранения данных PostgreSQL

Для проверки персистентности данных была создана тестовая таблица и добавлена запись.

SQL-команды:

```
CREATE TABLE test_data (
    id SERIAL PRIMARY KEY,
    text VARCHAR(100)
);

INSERT INTO test_data (text)
VALUES ('data saved');

SELECT * FROM test_data;
```

Результат:

```
 id |    text
----+------------
  1 | data saved
```

После этого контейнеры были перезапущены:

```
docker compose restart
```

Повторная проверка показала наличие ранее созданной записи.

Это подтверждает, что данные PostgreSQL сохраняются после перезапуска контейнеров благодаря использованию отдельного тома Docker.

Вывод:
 `screenshots/postgres.png`

## Остановка проекта

Остановка контейнеров:

```
docker compose down
```

Удаление контейнеров вместе с данными PostgreSQL:

```
docker compose down -v
```

Команду `docker compose down -v` следует использовать только при необходимости полного удаления данных базы данных.
